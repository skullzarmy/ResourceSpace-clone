ResourceSpace - Video tracks plugin configuration


This plugin allows users to create custom video files by using subtitle and audio track files that have been uploaded as alternative files for a video resource.

Because the processing of video files can take time the plugin utilises new job queue functionality so that the file creation can take place offline.  The plugin options page allows for a size limit to be configured and video files that are larger than this will be added to the queue. Once the processing is complete the user will be sent a system notification that the file is available

Setup instructions.

To enable the offline functionality the ‘$offline_job_queue’ configuration option must be enabled in config.php as below. A scheduled task/cron job should also be configured to run the pages/tools/offline_jobs.php as frequently as desired.

 $offline_job_queue=true;

  1. Enable the video_tracks plugin via Admin->System->Manage plugins->Asset processing
  2. Set the plugin configuration as required as below

This should be enabled if you want to be able to preview subtitles in the VideoJS player and is required because SRT files are no longer supported. Note that the previews will only work if MP4 video previews are used and will not work with FLV previews.

Export folder to save generated videos

Server path to a folder that the exported files will be saved to. The standard download filename will be used with an optional timestamp to prevent overwriting.

Available file output options. These should be tested on the server to ensure the syntax is correct for your installation of ffmpeg/avconv

This requires some technical knowledge. This allows you to construct the basic ffmpeg/avconv command syntax that will be used to create the custom videos from the selected inputs. You can set these up to meet your requirements and assign friendly names to them e.g. ‘Website preview’. It is advisable to test the command on the server to make sure you are happy with the results.
